# 6. `CONTRIBUTING.md` — Guia de Contribuição

> Objetivo: garantir **consistência, qualidade e previsibilidade**.

```md
# Guia de Contribuição

## Como Contribuir
- Fork do projeto
- Criação de branch
- Submissão de Pull Request

## Padrões de Commit
Formato esperado:
````

tipo(escopo): descrição curta

```

Exemplos:
- feat(auth): adicionar JWT
- fix(db): corrigir charset

## Padrões de Código
- Estilo
- Convenções
- Regras de lint

## Testes
- Testes obrigatórios
- Cobertura mínima esperada

## Revisão de Código
- Critérios de aprovação
- Checklist de PR

## Reportando Bugs
- Informações necessárias
- Formato recomendado
```
